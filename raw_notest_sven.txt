Response Aware of caching control per property
Developers can define the cache invalidation easily (Currently: TTL, LRU, LFU and Composite with any of them)
Business-aware cache rules (Data-Type Based Invalidation Strategy)
Vendor availability: 30 second TTL + push notifications (real-time critical)
Menus: 6-hour TTL + background refresh (staleness acceptable)
Cart: Immediate write-through (strong consistency required)
Orders: 10-second polling when active (contextual freshness)
Incientive: 30-minute TTL (staleness acceptable)
Static content (Static Vendor Data): Version-based, invalidate on deploy (rarely changes)

It should support multiple cache strategies per cache type (cache first, fallback) for the same set of data
It should have backwards compatibility, as losing all the cache is impacting our Screen performance and Screen availability, which is impacting the CVR and GMV
